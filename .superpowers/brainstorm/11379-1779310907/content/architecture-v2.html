How the Tool Works

Three stages: user submits a URL, the backend processes it, the result appears side-by-side.

1

You Submit

Paste an article URL and click submit. A Turnstile check prevents bots.

2

Backend Processes

Fetches the article, finds matching data points from our report library, sends both to AI for enrichment.

3

Compare Results

Original text on the left, enriched text on the right. Every new data point links to its source report.

What happens inside the backend (step 2):

Validate Check rate limit (10/hr), word count (max 5K), Turnstile passes
Fetch Download the article text from the URL you provided
Match Search pre-extracted data points for facts relevant to the article
Enrich Send article + matched data to DeepSeek V4 Flash (10K token cap)
Return Send back the enriched version with source citations

Before anything else runs (one-time setup):

Each source report in reports/ is processed once to extract discrete data points into reports/data-points.json. This is a build step — not repeated per request.
